Texture Editor 要件整理メモ

目的
- 現在の RoadEditor は「道を引く」ことに特化している。
- 今後は道路そのものではなく、道路に囲まれた領域や建物周辺を含む「マップ全体の質感設計」へ拡張したい。
- そのため、RoadEditor とは別に、地面レイヤーや領域テクスチャを扱える TextureEditor を用意する。

狙い
- 道路と道路の間の領域を、用途に応じた地面素材で埋める。
- 建物前、広場、草地、踏み固め地面などを手早く配置できるようにする。
- 境界が矩形や多角形のベタ貼りに見えないよう、ノイズやフェードで自然に崩す。

前提
- 3D上では薄い ground overlay mesh を重ねて表現する。
- UI は既存の RoadEditor と同じく Rect ベースの独自 UI を使う。
- 最初は高機能な地形エディタではなく、「地面テクスチャを置くための軽量エディタ」とする。


====================================
1. TextureEditor の役割
====================================

TextureEditor で扱うもの
- 草地レイヤー
- 土レイヤー
- 石畳 / レンガ / 広場レイヤー
- 建物前パッチ
- 道路接続パッチ
- 汚れ / 踏み跡 / デカール系の地面オーバーレイ

RoadEditor との分担
- RoadEditor:
  - 道路中心線の編集
  - 幅調整
  - 道路マテリアル調整
  - scatter 配置
- TextureEditor:
  - 道路外の面の質感構築
  - 領域塗り
  - 建物前や広場の演出
  - 地面境界の自然化

将来的な連携
- RoadEditor の道路データを TextureEditor が参照できるようにする。
- 将来的には「道路で囲まれた領域」を抽出し、その候補領域を TextureEditor 側で塗れるようにする。
- ただし初期実装では自動領域抽出は必須にしない。


====================================
2. 初期実装の最小スコープ
====================================

最小実装で必要な機能
- テクスチャレイヤーの追加
- 追加したレイヤーの位置、サイズ、回転の編集
- レイヤーごとのテクスチャ切り替え
- 色味補正（tint）
- 表示順の制御
- 保存 / 読み込み

最初に扱う形状
- 矩形
- 回転矩形

初期実装で後回しにしてよいもの
- 自由形状ポリゴン
- 道路囲み領域の自動生成
- 複雑なブーリアン演算
- 本格的なブラシ塗り

理由
- まずは「手置きで十分見栄えが上がる」状態を作ることが優先。
- 矩形を複数枚重ねるだけでも、建物前・広場・草パッチの表現はかなり改善できる。


====================================
3. Texture Layer データ要件
====================================

1レイヤーが持つ情報
- id
- label
- texture path
- category
  - grass
  - dirt
  - plaza
  - brick
  - stone
  - decal
- position (x, z)
- size (width, height)
- rotation
- yOffset
- tint (r, g, b, a)
- tiling scale
- softness
- edge noise amount
- edge noise frequency
- edge noise seed
- visible
- selected
- sort order

補足
- 初期段階では position は地面上の 2D 座標で十分。
- 高さは ground に沿う前提で固定でもよい。
- yOffset は Z-fighting 回避用に小さく持つ。

推奨カテゴリ
- BaseGround: 全体草地など
- Plaza: 広場やレンガ敷き
- Transition: 土帯や踏み固め境界
- Detail: 汚れ、轍、焦げ、湿り

4. UI 要件
====================================

基本方針
- RoadEditor と同様に、画面右側または左側の固定パネルで操作する。
- タブ構成にしてもよいが、初期段階では独立エディタでもよい。

必要な UI セクション
- Layer List
  - 追加
  - 複製
  - 削除
  - 上へ / 下へ
  - 表示ON/OFF
- Placement
  - 位置
  - サイズ
  - 回転
- Material
  - テクスチャ選択
  - tint
  - tiling scale
  - alpha
- Edge
  - softness
  - noise amount
  - noise frequency
  - seed
- Save / Load

最低限ほしい操作
- マウスで配置
- ドラッグで移動
- ハンドルまたはスライダーでサイズ変更
- 回転操作
- 選択中レイヤーの複製
- Undo

ショートカット案
- T: TextureEditor ON/OFF
- Delete: 選択レイヤー削除
- Ctrl+D: 複製
- Ctrl+Z: Undo
- Ctrl+S: Save
- L: Load


====================================
5. 境界線を自然にする要件
====================================

目的
- テクスチャ領域が「四角い板を置いただけ」に見えるのを避ける。
- 草と広場、土と道路の切り替えを自然にする。

必要な表現
- 外周フェード
- 外周のノイズ変形
- 細い中間帯

最低限の実装順
1. softness による外周フェード
2. noise による境界のぐにゃぐにゃ化
3. transition layer の追加

A. 外周フェード
- 領域の端ほど alpha を落とす。
- 矩形境界をそのまま見せないための最小機能。
- まずは内側 100%、外側に向けて減衰する簡易マスクでよい。

B. 境界ノイズ
- 境界線の半径または edge distance に対し noise を加算して揺らす。
- 完全ランダムではなく、seed 固定で再現性を持たせる。
- noise amount が 0 なら通常の綺麗な矩形として描画される。

境界ノイズの考え方
- base edge = 矩形境界までの距離
- warped edge = base edge + noise(samplePos * frequency + seed) * amount
- alpha = smoothstep(..., warped edge)

必要パラメータ
- edgeNoiseAmount
- edgeNoiseFrequency
- edgeNoiseSeed
- edgeSoftness

期待する見た目
- 草地の端が少し欠ける
- 広場の角がわずかに崩れる
- 建物前の土パッチが人工物感を減らす

注意
- noise を強くしすぎると俯瞰で輪郭が汚くなる。
- 初期値は弱めにする。
- 広場系は弱ノイズ、土や踏み固め帯は中ノイズ、汚れデカールは強ノイズが向く。


====================================
6. 描画方式の要件
====================================

初期実装案
- OneSidedPlane ベースの overlay mesh を使う。
- 各レイヤーを 1枚の平面として描画する。
- Transformer3D で位置、サイズ、回転を与える。

境界ノイズの実装候補
候補A: テクスチャ生成方式
- CPU で layer 用マスクテクスチャを生成する。
- その中でフェードとノイズ境界を描く。
- 実装しやすい。
- 初期実装向き。

候補B: シェーダ方式
- シェーダで edge distance と noise を使い alpha を決める。
- 拡張性が高い。
- 将来的にこちらへ移行してもよい。

初期推奨
- まずは CPU 生成テクスチャまたは既存のマテリアル生成系の拡張で実装する。
- RoadMaterialBuilder の考え方を TextureEditor 用に転用できるとよい。

描画順
- BaseGround
- Plaza / Dirt / Brick
- Transition
- Decal
- Road
- Scatter / 建物

または
- Ground base
- Road shoulder / road
- 地面 detail overlay

どちらが自然かは画面を見ながら調整する。


====================================
7. 保存データ要件
====================================

保存すべき内容
- レイヤー一覧
- 各レイヤーの transform
- texture path
- tint
- edge parameters
- sort order
- category
- editor settings

保存ファイルの考え方
- roads.toml とは別ファイルにする。
- 例:
  - data/ground_layers.toml
  - data/map_texture_layers.toml

必要性
- RoadEditor の保存と責務を分離するため。
- 将来的な自動生成やテンプレート化をやりやすくするため。

====================================
10. 最小完成条件
====================================

以下を満たせば初期版として成立
- 草地レイヤーを1枚敷ける
- 建物前に広場パッチを置ける
- 道路脇に土パッチを置ける
- 各パッチの位置 / サイズ / 回転を編集できる
- Save / Load できる
- 境界に softness をかけられる
- 境界に弱い noise をかけて不整形にできる

この段階で、見た目は
「道路しかない空間」から
「用途を持つ地面が存在するマップ」
へ進める。


====================================
11. 実装時の注意点
====================================

- 最初から自動化しすぎない。
- 人が少枚数を手で置いて気持ちよく調整できることを優先する。
- ノイズは強すぎると汚く見えるため、初期値はごく弱くする。
- 草、広場、土で同じノイズ量にしない。
- 矩形1枚で完成を目指さず、複数枚の重ね置きを前提にする。
- 道路と地面の境界は、いきなり完全解決を目指さず、まず transition layer で繋ぐ。


====================================
12. 結論
====================================

必要なのは「道路エディタの次」に来る「面のエディタ」である。
その中核機能は以下の2つ。

1. テクスチャレイヤーを簡単に追加・編集できること
2. 境界をフェード + ノイズで自然に崩せること

まずは軽量な TextureEditor を作り、矩形ベースの地面パッチ配置から始める。
その後、境界ノイズと保存機能を整えれば、マップ全体の見た目改善に直結する。
