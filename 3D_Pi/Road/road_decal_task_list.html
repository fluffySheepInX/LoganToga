道路端デカール / 小物配置 UI タスク一覧

対象: 3D_Pi / RoadEditor。目的は「道路と地面の境界を馴染ませる」ために、草・小石・泥・路肩素材を道路文脈に沿って配置できるようにすること。

Road Editor UX First Environment Art

1. 前提整理

  • 現状の主要課題は、道路内部の模様量ではなく 道路と地面の境界の遷移不足。
  • 改善には道路本体に加え、路肩・草・小石・泥 の周辺情報が必要。
  • そのため、単なる道路マテリアル UI だけでなく、デカール / 小物配置 UI が必要。
  • ただし UX 上、道路作成フローと小物配置フローは混ぜない。モード分離が前提。

2. 実装ゴール

アート目標

  • 道路端に移行帯を作る
  • 路肩に草・小石・汚れを配置できる
  • 交差点や建物前だけ密度を変えられる

UX目標

  • 「道路編集」と「装飾配置」を分離
  • 現在の選択対象が分かりやすい
  • プレビュー・削除・やり直しが明快

技術目標

  • 配置データを TOML 保存
  • アセット種別ごとに描画・配置ルールを分離
  • 将来の自動散布へ拡張可能にする

3. 必要な機能ブロック

機能 内容 優先度
Placement Mode 道路編集と別の配置モード。草 / 小石 / 泥デカール / 小物を切替。 A
Asset Palette 配置可能アセット一覧。カテゴリ、サムネイル、種別名を表示。 A
Placement Parameters サイズ、回転、密度、ランダム性、路肩優先度など。 A
Selection / Delete 置いた要素の選択、再配置、削除。 A
Brush Placement 1個置きではなく、ブラシで複数散布。 B
Rule-based Scatter 道路端・交差点・建物前に応じて自動的に偏らせる。 B
Preset System 「乾いた土道」「湿った路肩」などの配置プリセット。 C

4. データ構造タスク

  1. Asset definition を作る
    • id
    • displayName
    • category : grass / pebble / dirtDecal / prop
    • resourcePath
    • placementType : decal / billboard / mesh
    • defaultScaleMin, defaultScaleMax
    • alignToGround, randomYaw
  2. Placed instance を作る
    • assetId
    • position
    • rotation
    • scale
    • tint
    • layer / priority
    • optional: roadId, local distance to road edge
  3. Placement settings を作る
    • activeCategory
    • activeAssetId
    • brushRadius
    • density
    • jitter
    • snapToRoadShoulder
    • edgeBias
  4. Persistence を TOML に追加
    • [[placementAssets]] は不要。定義はコード or 別 TOML で保持。
    • [[placements]] で配置インスタンスを保存。
    • [placementSettings] で現在の UI 状態を保存。

5. UI/UX タスク

推奨タブ構成

  • Edit : 道路形状編集
  • Material : 道路材質調整
  • Scatter : 草・小石・泥・小物配置

Scatter タブの UI セクション

  1. Category
    • Grass
    • Pebble
    • Dirt / Mud Decal
    • Props
  2. Asset Picker
    • 名前だけでなく簡易プレビューを表示
    • 現在選択中を明確にハイライト
  3. Placement Mode
    • Single
    • Brush
    • Erase
    • Select
  4. Brush Controls
    • Brush Radius
    • Density
    • Random Rotation
    • Scale Range
    • Road Edge Bias
  5. Context Rules
    • Only near road edge
    • Avoid center lane
    • Boost near intersections
    • Boost near buildings

UX注意点

  • 道路編集と配置を同時に有効化しない。
  • 配置前に ゴーストプレビュー を出す。
  • 消去モードは色とカーソル形状を変えて誤操作を防ぐ。
  • 選択中オブジェクトはアウトライン or ハイライトで判別可能にする。
  • 1クリック1配置だけでなく、後でブラシ配置に進化できる構造にする。

6. 配置ロジック タスク

  1. 地面へのレイキャスト位置取得
  2. 道路中心線への最近接点取得
  3. 道路からの横距離を使って road edge band を計算
  4. その帯域に応じて草・小石・泥の置きやすさを変える
  5. Single placement を先に実装
  6. 次に Brush placement を追加
  7. 最後に rule-based scatter を追加

推奨ルール

  • Grass: 路肩外側に強く、道路中央は禁止
  • Pebble: 路肩と交差点にやや強く
  • Dirt decal: 道路端と交差点に強く
  • Small prop: 建物前 / 分岐周辺に限定

7. 描画タスク

種別 描画方式 用途
Grass Billboard / simple card 道路端の輪郭を壊す
Pebble Decal or tiny mesh cluster 路肩の粒感・遷移帯
Dirt / Mud Decal 地面と道路の中間層
Small Prop Mesh 生活感・物語性

8. アセット最低セット

  • 草アルファ 2種（短草 / 枯れ草）
  • 小石・砂利デカール 1〜2種
  • 泥・汚れデカール 1〜2種
  • 草混じり土タイル 1種
  • 小さな岩片メッシュ 1〜2種

9. フェーズ別タスク

Phase 1 - 基盤

  1. 配置データ構造追加
  2. TOML 保存/読込追加
  3. Scatter タブ UI 追加
  4. Single placement 実装
  5. Select / Delete 実装

Phase 2 - 実用化

  1. Grass / Pebble / Dirt のカテゴリ切替
  2. Ghost preview
  3. Brush placement
  4. Scale / rotation randomization
  5. 簡易 edge bias

Phase 3 - AAA寄り強化

  1. 道路局所座標に基づく rule-based scatter
  2. 交差点強調ルール
  3. 建物前補正ルール
  4. プリセット保存
  5. 自動散布 + 手動仕上げの併用

10. 完了条件

  • 道路と地面の境界に草・小石・泥を配置できる
  • 配置物を選択・削除・保存・読込できる
  • 道路編集と配置編集が誤操作なく切り替えられる
  • 交差点・路肩の見た目を手動で整えられる
  • 将来の自動散布へ拡張しやすい構造になっている
