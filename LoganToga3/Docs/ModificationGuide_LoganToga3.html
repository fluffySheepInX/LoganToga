LoganToga3 改修ガイド

「どこに何があり、何を変えたい時にどこを見るか」を素早く判断するための案内です。代表的な改修例と、調査・実装の入口をまとめます。

対象: LoganToga3 形式: C++ / Siv3D / TOML 目的: 改修時の入口確認

まず見るべき全体像

LoganToga3 は、起動・シーン・更新・描画・入力・データ定義・エディタ UI が比較的はっきり分かれています。迷った時は、次の流れで追うと位置を特定しやすいです。

1. 起動とシーン遷移: LoganToga3/Main.cpp から TitleScene / BattleScene が登録されます。
2. バトルシーン: LoganToga3/Scenes/BattleScene.h が更新と描画の大枠を呼びます。
3. 更新処理: LoganToga3/App/AppUpdate.h が入力、ワールド更新、通知、環境音などを束ねます。
4. 描画処理: LoganToga3/App/AppRender.h がバトル描画とエディタ UI 描画を束ねます。
5. 実処理: LoganToga3/Systems がゲームロジック、LoganToga3/UI が表示・操作、LoganToga3/Data が定義読み込みを担当します。

代表的な改修例と対応場所

データ ユニットの性能や種類を変えたい

まず InfoUnit/UnitSample.toml を確認します。読み込み側は Data/Loaders/UnitDefLoader.h、定義の利用側は Data/UnitCatalog.h と Data/DefinitionStores.h です。

対応の流れ: TOML に項目が既にあればデータ修正だけで済みます。新しい項目を追加するなら、定義型、ローダ、利用する System/UI の順に追います。

データ スキルを追加・調整したい

実データは InfoSkill/skill.toml、読み込みと保存は Data/Loaders/SkillDef*.h 系、エディタは UI/SkillEditor*.h 系を見ます。

対応の流れ: 定義値の追加なら SkillDefLoaderCommon.h や SkillDefValueParsing.h、保存なら SkillDefPersistence.h、エディタ反映なら SkillEditorInput.* と SkillEditorDraw.* を確認します。

ロジック 攻撃・ダメージ・ターゲット挙動を変えたい

戦闘更新は Systems/CombatSystem.h、ダメージ処理は Systems/DamageSystem.h、対象選択は Systems/TargetingSystem.h を中心に見ます。

対応の流れ: 「誰を狙うか」は Targeting、「命中や攻撃処理」は Combat、「HP 変化や死亡」は Damage という分担で追うと迷いにくいです。

ロジック 移動や経路探索を変えたい

移動命令は Systems/BattleMoveOrders.h、移動更新は Systems/MovementSystem.h、経路探索は Systems/PathfindingSystem.h です。

対応の流れ: 入力で移動先を作る箇所、命令として保存される箇所、毎フレームの移動処理、障害物や地形の扱いを順番に見ます。

ロジック 建築や建築メニューを変えたい

建築定義は InfoBuildMenu/BuildMenu.toml、読み込みは Data/Loaders/BuildActionDefLoader.h、入力は Systems/BattleBuildInputSystem.h、実処理は Systems/BuildSystem.h です。

対応の流れ: 表示だけなら UI/BattleUiPanels.Build.h、クリックや配置操作なら BattleBuildInput、建築成立条件なら BuildSystem を確認します。

UI マップエディタのパネルやボタンを変えたい

エディタ全体は UI/MapEditor*.h 系です。レイアウトは *LayoutRects.h、描画は *Draw*.h、入力は *Input*.h を優先して探します。

対応の流れ: Rect ベース UI の方針に合わせ、まず表示矩形を LayoutRects で確認し、描画、入力の順に変更します。

UI カメラ操作や UI 上での誤操作を変えたい

カメラ操作は Systems/CameraInputSystem.h と UI/QuarterView.h を見ます。UI 上でカメラやマップ操作を止める判定は IsCursorOnMapArea が入口です。

対応の流れ: 新しいパネルを追加した場合、そのパネル上でカメラやマップ操作が動かないように、該当 Rect を IsCursorOnMapArea の除外判定へ追加します。

AI 敵 AI や攻撃ウェーブを変えたい

AI 定義は InfoAI/AiProfiles.toml、読み込みは Data/Loaders/AiProfileDefLoader.h、実行は Systems/EnemyDirector*.h 系です。

対応の流れ: プロファイル値の調整で済むかを先に確認し、ロジック変更が必要なら Construction、Spawn、AttackWave、Tactics など目的別ファイルを見ます。

データ 資源ノードや資源定義を変えたい

資源定義は InfoResource/Resources.toml、読み込みは Data/Loaders/ResourceDefLoader.h、ロジックは Systems/ResourceSystem.h、エディタは UI/MapEditorResource*.h です。

対応の流れ: 資源の種類や表示は Data/UI、占領や取得の挙動は ResourceSystem、旗などの見た目は AppRuntimeState と renderer 側を確認します。

UI 音楽管理やプレビューを変えたい

音楽設定は Data/MusicSettings.h、再生制御は Data/MusicManager.h、プレビューは Data/MusicPreview.h を見ます。

対応の流れ: LoganToga3 では音楽エディタを特定画面専用ではなく、タイトル・戦闘・将来の設定画面など全体向けの基盤として扱います。

改修前の確認手順

  1. データ変更で済むか確認する。 ユニット、スキル、AI、資源、建築はまず TOML を見ます。
  2. 入口を確認する。 毎フレーム処理なら AppUpdate.h、描画なら AppRender.h、シーン単位なら Scenes を見ます。
  3. 責務ごとのファイルを追う。 入力は *Input*、表示は *Draw*、矩形は *LayoutRects*、保存は *Persistence* が目印です。
  4. UI 追加時はマウス除外判定を確認する。 新しいパネルを作ったら、カメラやマップクリックを止める必要があるか確認します。
  5. TOML 保存時は既存方針に合わせる。 LoganToga3 の TOML 書き出しは、全文を組み立てて必要最小限の改行だけを明示的に入れる方針です。
注意: LoganToga3 のエディタ UI は Rect ベースのボタン・パネルを基本にします。新しいエディタ UI を作る場合は SimpleGUI ではなく、既存の RectUiHelpers.h や LayoutRects 系に寄せてください。

ファイル名から役割を推測するコツ

*LayoutRects.h

画面上の矩形、パネル、ボタン位置を返すファイルです。クリック判定と描画の基準になります。

*Draw*.h

描画担当です。表示される文字、色、枠、リスト、アイコンなどを調整します。

*Input*.h

クリック、キー入力、ドラッグ、右クリックメニューなどの操作を処理します。

*Loader.h

TOML など外部データから定義を読み込む処理です。データ項目追加時の入口です。

*Persistence.h

保存担当です。エディタで変更した内容をファイルへ戻す時に確認します。

*System.h

ゲームロジック担当です。毎フレームの挙動、判定、状態更新を扱います。

具体例: 新しいエディタパネルを追加する時

  1. UI/MapEditorToolbar*.h でツールバー上の入口ボタンを検討します。
  2. パネルの矩形を UI/*LayoutRects.h に追加します。
  3. 表示処理を UI/*Draw*.h に追加します。
  4. クリックや入力処理を UI/*Input*.h に追加します。
  5. パネル上でカメラやマップクリックが動かないよう、必要なら Systems/CameraInputSystem.h の IsCursorOnMapArea に除外判定を追加します。
  6. 保存が必要なら Data または該当 UI/*FileOps* / *Persistence* 系に寄せます。

具体例: 新しい定義項目を TOML に追加する時

  1. 対象の実データ TOML を App/000_Warehouse/000_DefaultGame/070_Scenario/Info* から探します。
  2. 対応する定義型を Data または関連ヘッダで探します。
  3. 読み込み処理を Data/Loaders/*Loader.h に追加します。
  4. エディタで編集するなら UI/*EditorInput* と UI/*EditorDraw* に項目を追加します。
  5. 保存するなら *Persistence* や *FileOps* 系に追加します。
  6. 実際の挙動へ反映するなら Systems 側で定義値を参照します。

迷った時の判断基準

症状 判断
画面には出ているが押せない 描画ではなく、Input 側か Rect の不一致を疑います。
押せるが見た目が違う Draw 側、アイコンパス、フォント、色定義を疑います。
TOML を変えても反映されない Loader が読んでいる項目名、DefinitionStores への格納、再読み込みタイミングを確認します。
エディタでは変わるがゲームに反映されない エディタ状態から BattleWorld / DefinitionStores へ同期する処理を確認します。
UI 上で意図しないマップ操作が起きる 対象パネルの Rect が IsCursorOnMapArea の除外に含まれているか確認します。
